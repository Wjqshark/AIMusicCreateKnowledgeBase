# 古典音乐编曲子库的收录建议

这部分是专门给你的 AI 音乐知识库落地用的。

## 目标

把“古典音乐编曲学习资料”转成：

- 可索引
- 可拆解
- 可被 LLM 读取
- 可用于生成 style prompt / 编曲建议 / 结构建议

## 每种知识文件建议统一格式

```md
---
id: classical_orchestration_strings_001
type: orchestration
title: 弦乐组基础写法
tags: [classical, orchestration, strings]
related:
  - classical_orchestration_layering_001
  - classical_romantic_orchestral_prompt_001
version: 1.0.0
---

# 适用范围
# 核心规则
# 常见错误
# 示例
# 可抽象为 AI 提示词的字段
# 关联资源
```

## 最适合先落库的内容

先做这 8 个知识文件，收益最高：

1. `functional_harmony.md`
2. `voice_leading.md`
3. `strings.md`
4. `woodwinds.md`
5. `brass.md`
6. `orchestral_layering.md`
7. `sonata_allegro_form.md`
8. `romantic_orchestral.md`

## 为什么这么拆

因为古典音乐相关的“编曲”并不是单靠一个 style prompt 就能学会，它依赖：

- 和声
- 声部进行
- 乐器特性
- 曲式结构
- 总谱经验

把它拆开，LLM 才更容易按需求组装输出。
